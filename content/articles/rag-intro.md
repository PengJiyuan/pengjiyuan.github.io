## RAG 定义

**RAG = Retrieval Augmented Generation = 检索增强生成**

简单说：让 AI 能"查资料"后再回答问题。

---

## 为什么需要 RAG？

### AI 的问题

| 问题 | 说明 |
|------|------|
| 知识有限 | AI 训练数据有截止日期 |
| 可能编造 | 会一本正经胡说八道 |
| 成本高 | 微调模型很贵 |

### RAG 的解决方案

```text
用户问题 → 检索相关资料 → 把资料给 AI → AI 生成准确回答
```

---

## RAG 工作流程

```text
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   用户问题   │ → │  检索资料   │ → │  构建提示词  │ → │  AI 生成回答 │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                          ↑
                   ┌──────┴──────┐
                   │  向量数据库  │
                   │ (知识库)     │
                   └─────────────┘
```

### 1. 知识库准备
- 准备文档（PDF、网页、文本等）
- 把文档切分成小块（chunks）
- 用 Embedding 模型把文字转成向量
- 存到向量数据库

### 2. 检索
- 把用户问题转成向量
- 在向量数据库中找相似的文档

### 3. 生成
- 把找到的资料 + 用户问题一起给 AI
- AI 根据资料生成回答

---

## RAG 的优势

| 优势 | 说明 |
|------|------|
| 知识新鲜 | 可以接入最新资料 |
| 减少幻觉 | 基于真实资料回答 |
| 成本低 | 不用重新训练模型 |
| 可追溯 | 能看到引用来源 |
| 可更新 | 直接更新知识库即可 |

## RAG 应用场景

| 场景 | 说明 |
|------|------|
| 企业知识库 | 员工问答系统 |
| 客服机器人 | 基于产品文档回答 |
| 论文助手 | 论文摘要、问答 |
| 医疗助手 | 医学知识库问答 |

---

## RAG 核心技术

### 1. 文本分块 (Chunking)
- 把长文档切成小段落
- 常用方法：固定大小、重叠滑动

### 2. 向量嵌入 (Embedding)
- 常用模型：BGE、OpenAI Embedding、Jina Embedding

### 3. 向量数据库
- Pinecone、Milvus、Chroma、Qdrant

### 4. 相似度检索
- 余弦相似度 (Cosine Similarity)
- 点积 (Dot Product)

---

## 2026 年 RAG 发展趋势

- **多模态 RAG** - 支持图片、视频检索
- **原生 RAG** - RAG 与模型深度整合
- **Graph RAG** - 用知识图谱增强检索
- **实时 RAG** - 动态更新知识库
